## Types

```typescript
interface Bar {
  date: string;       // ISO date, e.g. "2026-01-15"
  open: number;
  high: number;
  low: number;
  close: number;
}

type SignalType = "BUY" | "EXIT";

interface Signal {
  type: SignalType;
  sourceBarIndex: number;  // index of the bar whose close generated this signal
}

interface Position {
  entryBarIndex: number;
  entryDate: string;
  entryPrice: number;
  stopPrice: number;
  size: number;            // whole units, rounded down
  riskPerUnit: number;      // entryPrice - stopPrice, captured at entry
  riskAmount: number;       // riskPerUnit * size, captured at entry
}

interface Trade {
  entryDate: string;
  entryPrice: number;
  exitDate: string;
  exitPrice: number;
  size: number;
  exitReason: "SIGNAL" | "STOP";
  pnl: number;              // absolute currency
  rMultiple: number;        // pnl / riskAmount
}

interface Account {
  equity: number;           // current account equity
  openPosition: Position | null;
}

interface BacktestConfig {
  fastLength: number;        // default 10
  slowLength: number;        // default 20
  riskPercent: number;       // default 0.01 (1%)
  atrLength: number;         // default 14
  atrStopMultiple: number;   // default 2
  startingEquity: number;
}

interface BacktestSummary {
  totalTrades: number;
  winRate: number;           // wins / totalTrades, 0 if totalTrades === 0
  netPnl: number;            // sum of all trade.pnl
}

interface BacktestResult {
  trades: Trade[];
  summary: BacktestSummary;
  finalEquity: number;
}
```

## Pure functions

- `sma(values: number[], length: number, index: number): number | null` — simple moving average of `values[index - length + 1 ..= index]`; returns `null` if fewer than `length` values are available up to `index`.
- `atr(bars: Bar[], length: number, index: number): number | null` — Average True Range (Wilder/simple average of True Range over `length` periods ending at `index`); returns `null` if insufficient bars.
- `detectCrossover(prevFast: number, prevSlow: number, currFast: number, currSlow: number): SignalType | null` — returns `"BUY"` if fast crosses from at-or-below to strictly above slow, `"EXIT"` if fast crosses from at-or-above to strictly below slow, else `null`.
- `positionSize(equity: number, riskPercent: number, entryPrice: number, stopPrice: number): number` — computes `floor((equity * riskPercent) / (entryPrice - stopPrice))`.
- `computeStopPrice(entryPrice: number, atrValue: number, atrStopMultiple: number): number` — computes `entryPrice - (atrStopMultiple * atrValue)`.
- `isStopHit(bar: Bar, stopPrice: number): boolean` — returns `true` if `bar.low <= stopPrice`.
- `pnl(entryPrice: number, exitPrice: number, size: number): number` — computes `(exitPrice - entryPrice) * size`.
- `rMultiple(pnlValue: number, riskAmount: number): number` — computes `pnlValue / riskAmount`.
- `isWin(pnlValue: number): boolean` — returns `true` if `pnlValue > 0` (strictly).
- `summarize(trades: Trade[]): BacktestSummary` — aggregates total trade count, win rate (using `isWin`), and net P&L.

## Engine

```typescript
function runBacktest(bars: Bar[], config: BacktestConfig): BacktestResult
```

Iterates `bars` in chronological order, maintaining one `Account` (starting `equity = config.startingEquity`, `openPosition = null`), computing SMA/ATR series incrementally, detecting signals via `detectCrossover`, applying the fill and stop rules below, and closing out `Trade` records into the returned `trades` array. Returns `{ trades, summary: summarize(trades), finalEquity: account.equity }`.

## Rules encoded

- **Warmup rule**: No signal is evaluated until at least `slowLength` (20) bars of closes are available (i.e., `sma(closes, slowLength, index)` is non-null). Bars before this return no signal.
- **Crossover rule (buy)**: A `BUY` signal is generated at bar `i` when `fastSMA[i-1] <= slowSMA[i-1]` and `fastSMA[i] > slowSMA[i]`.
- **Crossover rule (exit)**: An `EXIT` signal is generated at bar `i` when `fastSMA[i-1] >= slowSMA[i-1]` and `fastSMA[i] < slowSMA[i]`.
- **Fill convention (NEXT_BAR_OPEN)**: A signal generated from bar `i`'s close is filled at bar `i+1`'s open. If bar `i+1` does not exist (end of data), the signal is dropped. This applies to both entries and signal-based exits — never to stop-loss exits.
- **One-position rule**: If `account.openPosition !== null`, any `BUY` signal occurring while it is open is ignored (no pyramiding, no reversal).
- **Position sizing rule**: On a filled `BUY`, `stopPrice = computeStopPrice(fillPrice, atr(bars, config.atrLength, entryBarIndex), config.atrStopMultiple)`; `size = positionSize(account.equity, config.riskPercent, fillPrice, stopPrice)`. `riskAmount = (fillPrice - stopPrice) * size`, fixed at entry and not recalculated during the life of the trade.
- **Default risk constant**: `DEFAULT_RISK_PERCENT = 0.01` (1% of current equity), used when `config.riskPercent` is not overridden.
- **Stop-loss definition constant**: `DEFAULT_ATR_LENGTH = 14`, `DEFAULT_ATR_STOP_MULTIPLE = 2`. Stop price = `entryPrice - (2 * ATR14)` at entry time, fixed for the life of the trade (not trailed).
- **Stop-hit same-bar fill rule**: On each bar while a position is open, if `isStopHit(bar, position.stopPrice)` is true, the position is closed at exactly `position.stopPrice` on that same bar (`exitReason: "STOP"`), taking priority over any signal-based exit evaluated for that same bar.
- **Win definition**: A trade counts as a win only if `pnl > 0` strictly; `pnl === 0` is not a win (rule for `isWin`).
- **Purity constraint**: `sma`, `atr`, `detectCrossover`, `positionSize`, `computeStopPrice`, `isStopHit`, `pnl`, `rMultiple`, `isWin`, and `summarize` accept only explicit inputs (arrays/primitives) and perform no I/O, no date fetching, no mutation of arguments.
